# 📜 Kelola Sertifikat

Fitur **Sertifikat** memungkinkan admin untuk membuat dan menerbitkan sertifikat digital secara otomatis kepada peserta maupun panitia yang mengikuti suatu agenda. Sertifikat didesain menggunakan **Certificate Designer** — sebuah editor visual drag-and-drop yang fleksibel.

## 1. Membuka Certificate Designer

1. Buka halaman **Detail Agenda** pada agenda yang ingin Anda buatkan sertifikatnya.
2. Cari bagian **Sertifikat** di halaman pengaturan agenda.
3. Anda akan melihat area **Certificate Designer** dengan panel *Tools* di sisi kiri dan area *Preview* di sisi kanan.

## 2. Upload Template PDF

Langkah pertama adalah menyiapkan desain latar belakang sertifikat dalam format PDF.

1. Klik tombol **Upload Template (PDF)** di bagian atas area Preview.
2. Pilih file PDF yang sudah Anda desain sebelumnya (misalnya dari Canva, Adobe Illustrator, atau CorelDRAW).
3. Sistem akan otomatis mendeteksi dimensi PDF dan menampilkan pratinjau gambar dari template tersebut.
4. Aktifkan toggle **Active** agar konfigurasi sertifikat ini aktif dan bisa dicetak.

::u-alert{variant="solid" color="info"}

# title

Tips Desain Template

# description

Pastikan template PDF Anda menyisakan area kosong untuk elemen-elemen dinamis seperti nama peserta, tanggal, QR Code, dan tanda tangan. Elemen-elemen ini akan ditambahkan melalui Designer.
::

## 3. Menambahkan Elemen ke Sertifikat

Pada panel **Tools**, tersedia beberapa elemen yang bisa Anda tambahkan ke sertifikat:

| Elemen | Ikon | Fungsi |
| :--- | :---: | :--- |
| **Nama Peserta** | 👤 | Menampilkan nama lengkap penerima sertifikat secara otomatis. |
| **Peran** | 🪪 | Menampilkan peran penerima (Peserta / Panitia). |
| **Teks Statis** | ✏️ | Teks bebas yang bisa Anda tentukan isinya (misal: judul acara). |
| **Tanggal** | 📅 | Menampilkan tanggal pelaksanaan secara otomatis. |
| **QR Code Verifikasi** | 📱 | QR Code untuk memvalidasi keaslian sertifikat. |
| **Kode Sertifikat** | 🔑 | Nomor unik sertifikat (misal: `/II.3.AI/B01.01/01.A0/Sert/II/2026`). |
| **Tanda Tangan** | ✍️ | Area tanda tangan penandatangan sertifikat. |

### Cara menambahkan elemen:

1. Klik salah satu tombol elemen di panel **Tools**.
2. Elemen akan muncul di area Preview dengan posisi default.
3. **Drag** (seret) elemen untuk memindahkan posisinya.
4. **Resize** (ubah ukuran) menggunakan handle di pojok kanan bawah elemen.
5. Gunakan tombol **Tengahkan Horizontal** dan **Tengahkan Vertikal** pada toolbar konteks untuk menyejajarkan elemen.

## 4. Mengatur Properti Elemen

Klik pada elemen di area Preview untuk menyeleksinya. Panel **Properties** akan muncul di sisi kiri dengan pengaturan berikut:

- **Font Size**: Ukuran teks (dalam satuan pt).
- **Font Family**: Pilih antara *Times New Roman* atau *Helvetica / Arial*.
- **Color**: Warna teks.
- **Style**: Aktifkan **Bold** jika diperlukan.
- **Align**: Atur perataan teks — Kiri, Tengah, atau Kanan.

Untuk elemen khusus:
- **Teks Statis**: Isi konten teks di kolom *Text Content*.
- **Kode Sertifikat**: Masukkan format nomor sertifikat di kolom *No Sertifikat*.
- **QR Code**: Data QR akan otomatis terisi oleh sistem (URL verifikasi).

## 5. Konfigurasi Tanda Tangan

Elemen **Tanda Tangan** memiliki dua mode penandatangan:

### a. Eksternal (Upload Gambar TTD)

Mode ini digunakan untuk penandatangan dari **luar sistem** (misalnya Dekan, Dosen Pembimbing).

1. Pilih mode **Eksternal** pada panel Properties.
2. Klik **Upload TTD (PNG)** untuk mengunggah gambar tanda tangan.

   ::u-alert{variant="solid" color="warning"}

   # title

   Format Gambar

   # description

   Gambar tanda tangan **wajib berformat PNG** dengan latar belakang **transparan**. Tanda tangan harus berwarna hitam agar terlihat jelas di atas template.
   ::

3. Isi **Nama Penandatangan** (misal: *Prof. Dr. Ahmad Yani, M.T.*).
4. Isi **Jabatan Penandatangan** (misal: *Dekan Fakultas Teknik*).

### b. Member Sistem (Tanda Tangan Digital)

Mode ini digunakan untuk penandatangan **internal** yang terdaftar sebagai anggota di sistem HIMATIKA (misalnya Ketua Umum, Sekretaris Umum).

1. Pilih mode **Member Sistem**.
2. Cari dan pilih nama anggota pada dropdown **Member Penandatangan**.
3. Isi **Jabatan Penandatangan**.
4. Tanda tangan akan berupa **QR Code Kriptografis** yang di-generate secara digital oleh sistem.

::u-alert{variant="solid" color="info"}

# title

Perbedaan Kedua Mode

# description

- **Eksternal**: Tanda tangan berupa gambar statis yang sudah di-upload. Cocok untuk pihak di luar organisasi.
- **Member Sistem**: Tanda tangan berupa QR Code digital. Member yang ditunjuk harus melakukan konfirmasi tanda tangan secara manual melalui menu *Tanda Tangan* di dasbor mereka.
::

## 6. Pratinjau & Penyimpanan

- Semua perubahan yang Anda lakukan di Designer akan otomatis tersimpan (auto-save).
- Anda bisa melihat informasi **Dimensions** (ukuran template dalam satuan pts) dan **Template** (URL file yang di-upload) di bagian bawah area Preview.
- Pastikan toggle **Active** dalam keadaan aktif agar sertifikat bisa diterbitkan.

## 7. Penerbitan Sertifikat

Setelah desain selesai dan agenda telah berakhir:

1. Sistem akan men-generate sertifikat PDF untuk setiap peserta/panitia yang memenuhi syarat.
2. Jika menggunakan tanda tangan **Member Sistem**, sertifikat akan masuk ke antrian tanda tangan PIC terkait.
3. Setelah semua PIC menandatangani, sertifikat akan tersedia dan bisa diunduh oleh penerima melalui akun mereka masing-masing.
