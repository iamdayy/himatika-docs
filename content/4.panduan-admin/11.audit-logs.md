# 📋 Audit Logs

Fitur **Audit Logs** mencatat seluruh aktivitas penting yang terjadi di sistem HIMATIKA. Log ini berfungsi sebagai catatan transparan untuk memantau siapa yang melakukan apa dan kapan.

Akses: Sidebar Admin → **`Audit Logs`** atau Dasbor Admin → Tab **`Audit`**

---

## 1. Fungsi Audit Logs

Audit Logs membantu admin dalam:

- **Keamanan** — Mendeteksi aktivitas mencurigakan (login dari IP tak dikenal, perubahan data beruntun).
- **Transparansi** — Memberikan bukti digital bahwa semua perubahan data dilakukan secara sah.
- **Investigasi** — Menelusuri kronologi jika terjadi masalah atau kesalahan data.
- **Akuntabilitas** — Memastikan setiap tindakan admin tercatat.

---

## 2. Informasi yang Tercatat

Setiap entri log berisi:

| Field | Keterangan | Contoh |
|:---|:---|:---|
| **Action** | Jenis aktivitas yang dilakukan | `login`, `update_member`, `sign_document`, `create_agenda` |
| **User** | Pelaku aktivitas (anggota/admin) | *Ahmad Yani (2021100001)* |
| **IP Address** | Alamat IP asal aktivitas | `192.168.1.100` |
| **Details** | Informasi tambahan spesifik per aktivitas | Data yang berubah, ID dokumen, dll |
| **Target** | Objek yang terpengaruh | ID anggota, ID agenda, dll |
| **Timestamp** | Waktu kejadian | *13 Maret 2026, 20:30 WIB* |

---

## 3. Jenis Aktivitas yang Dicatat

Berikut beberapa kategori aktivitas yang dicatat oleh sistem:

| Kategori | Contoh Aktivitas |
|:---|:---|
| **Autentikasi** | Login berhasil/gagal, logout, registrasi akun baru |
| **Data Anggota** | Edit profil, validasi anggota, ganti email/password |
| **Agenda** | Membuat, mengedit, atau menghapus agenda |
| **Dokumen** | Generate surat, tanda tangan dokumen, download |
| **Konfigurasi** | Perubahan pada config sistem |
| **Sertifikat** | Generate dan tanda tangan sertifikat |

---

## 4. Retensi Data

::u-alert{variant="soft" color="info"}
#title
⏱️ Masa Simpan Log
#description
Log audit **disimpan selama 3 bulan** (90 hari) sejak tanggal pencatatan. Setelah itu, log akan dihapus secara otomatis oleh sistem untuk menjaga efisiensi penyimpanan. Jika Anda perlu menyimpan log lebih lama, gunakan fitur **Export Excel** sebelum log kadaluarsa.
::

---

## 5. Tips Penggunaan

- **Periksa secara berkala** — Luangkan waktu untuk memeriksa audit log setidaknya sekali seminggu.
- **Perhatikan login gagal** — Beberapa kali login gagal pada akun yang sama bisa mengindikasikan percobaan akses tidak sah.
- **Gunakan untuk pelaporan** — Log audit bisa menjadi lampiran dalam Laporan Pertanggungjawaban (LPJ) kepengurusan.
