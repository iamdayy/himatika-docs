## 1. HIMATIKA (Nuxt 3)

Repositori utama yang menjalankan seluruh aplikasi:

| Komponen | Teknologi | Fungsi |
|:---|:---|:---|
| **Frontend** | Vue 3 + Nuxt 3 | Antarmuka pengguna (halaman, komponen, layout) |
| **Backend API** | Nuxt Server API (Nitro) | REST API untuk autentikasi, CRUD data, dan logika bisnis |
| **Database** | MongoDB + Mongoose | Penyimpanan data anggota, agenda, dokumen, dll |
| **Auth** | Sidebase Auth | Manajemen sesi, JWT, OAuth (Google) |
| **Payment** | Midtrans Snap | Gerbang pembayaran (QRIS, VA, E-wallet) |
| **i18n** | Nuxt I18n | Dukungan multi-bahasa (Indonesia, English) |

### Struktur Direktori Utama

```
himatika/
├── app/
│   ├── pages/          # Halaman (routing otomatis)
│   ├── components/     # Komponen UI reusable
│   ├── composables/    # Logic hooks
│   ├── stores/         # State management (Pinia)
│   └── layouts/        # Template layout (client, dashboard)
├── server/
│   ├── api/            # API endpoints (35+ modul)
│   ├── models/         # Mongoose models (24 model)
│   ├── middleware/      # Auth & request middleware
│   └── utils/          # Helper functions
└── types/              # TypeScript type definitions
```

## 2. PDF Worker (Python/Flask)

Microservice terpisah yang menangani operasi berat terkait file:

| Komponen | Teknologi | Fungsi |
|:---|:---|:---|
| **Framework** | Flask + Flask-CORS | REST API server |
| **PDF Processing** | ReportLab + PyMuPDF + pypdf | Generate & manipulasi PDF |
| **Excel** | openpyxl | Import & export spreadsheet |
| **QR Code** | qrcode + zxingcpp | Generate & scan QR code |
| **Image** | Pillow (PIL) | Kompresi & manipulasi gambar |
| **Storage** | Cloudflare R2 (S3) | Upload & simpan file hasil |
| **Database** | PyMongo | Akses data MongoDB |

### Daftar Endpoint

| Endpoint | Method | Fungsi |
|:---|:---:|:---|
| `/api/sheet/export` | POST | Export data ke file Excel (.xlsx) |
| `/api/sheet/import` | POST | Import data dari file Excel |
| `/api/sign/process` | POST | Overlay tanda tangan QR ke PDF |
| `/api/pdf/search-text` | POST | Cari teks dalam PDF (untuk posisi TTD) |
| `/api/pdf/activiness-letter` | POST | Generate Surat Keterangan Aktif |
| `/api/pdf/ticket` | POST | Generate tiket agenda (PDF) |
| `/api/pdf/scan-qr` | POST | Scan QR Code dari file PDF |
| `/api/pdf/certificate-preview` | POST | Preview template sertifikat |
| `/api/pdf/certificate` | POST | Generate sertifikat final |
| `/api/tools/qr` | POST | Generate QR Code (base64) |
| `/api/tools/compress-image` | POST | Kompresi gambar (JPEG) |
| `/api/tools/upload-image` | POST | Upload gambar ke R2 Storage |

## 4. Alur Data Utama

### Generate Surat Keaktifan
```
Anggota klik "Generate"
    → Nuxt API membaca data member & poin
    → Kirim ke PDF Worker /api/pdf/activiness-letter
    → PDF Worker buat PDF → upload ke R2
    → URL PDF disimpan ke MongoDB
    → Dokumen masuk antrian tanda tangan
    → PIC menandatangani → Nuxt kirim ke PDF Worker /api/sign/process
    → PDF Worker overlay QR ke PDF → upload ke R2
    → Dokumen siap diunduh
```

### Generate Sertifikat
```
Admin desain template di Certificate Designer
    → Template diupload ke PDF Worker /api/pdf/certificate-preview
    → Preview image dikembalikan
    → Saat cetak: Nuxt kirim data ke PDF Worker /api/pdf/certificate
    → PDF Worker generate PDF → upload ke R2
    → Sertifikat masuk antrian tanda tangan (jika ada member signer)
```
